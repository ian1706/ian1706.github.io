BACKEND

Una página web está formada por dos capas muy importantes, las cuales permiten el acceso a la información solicitada, la capa superficial donde el usuario interactúa para solicitar datos llamada front-end, descrita de otra forma seria la parte visible de la web en la cual se encuentran los gráficos y la interacción con el usuario, pero al hablar de Back-end nos referimos a la parte invisible la cual es la encargada de la parte lógica y cumple con la función de acceder a la información en la base de datos, por lo que el backend trabaja en conjunto con el frontend para proporcionar al usuario final la información solicitada.
Entre las principales herramientas que se utilizan, están los frameworks los cuales son una especie de plantilla que sirven como punto de partida para la organización y desarrollo del software.

Los principales frameworks para el backend son:

  • Spring (Java)
  • Django (Python)
  • Laravel (Php)
  • Ruby on Rails (Ruby)

Spring (JAVA)

Java es uno de los lenguajes de programación más utilizados y solicitados por las empresas para crear aplicaciones de arquitectura cliente/servidor, por su alto nivel y sus múltiples librerías para realizar cualquier tipo de proyectos, pero a medida que la tecnología avanza y el aumento de la complejidad de los sistemas web tanto gráficamente, como en la seguridad, se volvió imprescindible realizar herramientas de ayuda para no tener que repetir código y reducir el tiempo y el espacio, de esta forma se han ido creando herramientas llamadas frameworks los cuales hacen más sencillas el desarrollo de aplicaciones.
Spring framework proporciona un modelo integral de programación y configuración para aplicaciones empresariales modernas basadas en Java, en cualquier tipo de plataforma de implementación y se basa en conceptos de inversión de control e inyección de dependencia, los mismos que actúan juntos para ayudar al ahorro de tiempo en la inicialización y la conexión con otros componentes, su aplicación es muy sencilla y liviana, ahorra el tiempo en configuración en xml y su velocidad de actualización es muy rápida.

Caracteristicas Ventajas Desventajas
  • Integración con otros softwares.
  • Permite realizar pruebas antes de finalizar la aplicación, simulacros de objetos, WebtestVlient, Framework TestContext
  • Facilita la programación basada en el Modelo Vista Controlador
  • Es un código de fuente más simplificado.
  • Posee una estructura base de alto rendimiento que no necesita modificarse demasiado
  • Proporciona una API para traducir excepciones específicas de la tecnología.
  • No podemos saber si se realizó bien la inyección de un objeto más que en el tiempo de ejecución
  • Es un framework complejo que requieren tener claros ciertos conceptos.
  • El contenedor no es ligero y no es recomendable utilizarlo en tiempo real ni en aplicaciones móviles.

Django (Python)

Es un framework de alto nivel, flexible y de código reutilizable, el cual permite desarrollar aplicaciones web seguros y pragmáticos además de su fácil mantenimiento, django fue diseñado para poder realizar cambios en una parte especifica de la aplicación sin afectar a las demás piezas de la misma por lo tanto este framework esta implementado en el modelo vista templeid (MVT), además interactúa con la base de datos de una manera muy practica

Caracteristicas Ventajas Desventajas
  • posee mecanismos para protegerla base de datos, formularios y JavaScript.
  • La interfaz de acceso a la base de datos y consultas es sumamente rápida
  • Se puede ejecutar en varias plataformas
  • Posee un ORM que abstrae la necesidad de realizar consultas SQL y soporta múltiples bases de datos
  • Posee un solido sistema de permiso para restringir el acceso a los datos dependiendo como el desarrollador lo requiera.
  • Posee una amplia variedad de librerías.
  • Posee una extensa documentación que puede ser confusa y diversa
  • Se debe tener un vasto conocimiento de Python
  • La implementación de sockets es algo compleja de usar.
